01 · Why now

企业级 AI 自动化窗口

信息碎片IM / 邮件 / JIRA / 表格 分散
协作高频跨团队需求多变
交付缺闭环AI 能力有了，生产系统缺位

目标：让高频动作可编排、可审计、可复用。

01 · 跨岗重复动作

相同场景在不同岗位反复出现

重复动作涉及岗位代价
信息同步 / 纪要 / 版本对齐产品、商务、HR、行政多版本冲突，遗漏责任人
数据整理 / 口径校验 / 报告产品、测试、运营、财务重复造表，解释口径耗时
流程问答 / SOP 辅导IT、法务、HR、客服高频答疑占用专家时间
工单 / Bug 收敛与路由开发、测试、运维误分拣、信息不全导致返工
01 · 归并场景

5 套跨岗复用工作流

信息同步官

多版本摘要 + 回执闭环，适用于产品/商务/HR/行政。

自动分析师

清洗→解释→汇报，适用于产品/测试/运营/财务。

研发 Copilot

检索→方案→代码→文档，支持开发/测试。

Bug Intake

聚类/评级/路由/补全，全自动 JIRA 流程。

会议 PMO

转写→纪要→行动项→提醒，全员可用。

02 · 场景 1

信息同步官：版本不一致、确认慢

  • 同一信息向多角色重复讲解，版本漂移。
  • 管理层/执行层/外部颗粒度不同，易失真。
  • 缺回执与留痕，无法确认阅读与理解。
后果

决策延迟、责任模糊、品牌口径冲突。

反馈周期 1–2 天
02 · Flow

多版本摘要 + 回执闭环

触发

会议/IM/邮件事件收敛。

上下文收集

链接、附件、时效性标签。

受众分群

管理层/执行层/外部分类。

多版本摘要

按模板生成 3 套口径。

推送

IM/邮件/群同步，附阅读要求。

回执

确认/疑问收集，写回记录。

留痕

版本存档，可审计与追踪。

02 · 节点演绎

Dify 节点与责任

触发器

会议日历 Webhook / 邮件钩子 / IM Slash 命令。

上下文汇聚

收集附件、链接、时效标签；落库保留原始上下文。

分群模板

分支节点：管理层/执行层/外部调用不同提示词模板。

推送与回执

工具节点发送 IM/邮件，回执写回数据表并生成留痕。

02 · 场景 2

自动分析师：数据整理与解释耗时

  • Excel/BI 导出口径不一致，清洗靠人工。
  • 图表与结论撰写重复，缺模板。
  • 异常缺少自动归因与提醒。
现状

周报/月报多人轮值，难以追溯口径与版本。

02 · Flow

清洗 → 解释 → 汇报

导入

Excel / SQL 结果 / BI 导出。

清洗校验

口径校验、缺失值补齐、格式标准化。

指标画像

生成核心指标概览。

异动检测

阈值/同比环比异常识别。

异动归因

找原因、给建议、列验证步骤。

报告生成

图表 + 结论模板化输出。

推送

IM/邮件/文档链接，写回版本。

02 · 节点演绎

Dify 节点与责任

数据入口

上传文件/SQL 结果/BI 导出；元数据记录来源与口径。

清洗校验

工具节点做正则/缺失补齐；失败写入错误日志。

LLM 解读

按模板生成指标解释/异常归因，内嵌 Guardrails。

报告模板

渲染图表与结论；版本化输出 + 回传 IM/邮件。

02 · 场景 3

研发 Copilot：知识难复用，定位耗时

  • 日志/报错定位靠经验，重复提问浪费时间。
  • 脚本/代码片段分散，缺统一入口与标准。
  • 变更说明与测试建议缺模板、不可审计。
痛点

新人 ramp-up 慢，核心知识沉淀不到位。

检索 → 方案 → 代码 → 文档

输入

文档

同步

02 · 节点演绎

Dify 节点与责任

检索增强

代码/文档/FAQ 检索；引用追踪，减少幻觉。

假设与验证

LLM 输出修复假设，工具节点生成验证 checklist。

代码节点

脚本/SQL/配置片段生成，附解释与安全提示。

文档同步

MR 描述、变更说明、测试建议自动写回仓库/Wiki。

02 · 场景 4

Bug Intake：提报/分拣/补全靠人工

  • 缺去重/聚类，重复 Bug 泛滥。
  • 严重级别与路由需要人工判断。
  • 复现步骤不完整，返工率高。
代价

误分拣导致延迟，测试/开发沟通成本高。

02 · Flow

自动分拣 + 补全 + 推送

收件

工单/反馈/JIRA Webhook。

去重聚类

相似度聚类，标记重复。

评级

严重级别、影响面评估。

路由

按模块/值班表自动分派。

补全

复现步骤、环境、日志缺口补齐。

创建/更新

写入 JIRA，附标签与负责人。

摘要推送

日报/提醒推送到群与责任人。

02 · 节点演绎

Dify 节点与责任

入口

Webhook/表单/IM 收集；原始描述与附件留存。

去重聚类

LLM + 相似度聚类，标记重复并归并。

评级路由

规则 + LLM 判定严重级别，按模块/值班表分派。

补全与推送

复现步骤补齐，调用 JIRA API 创建/更新，推送日报。

02 · 场景 5

会议 PMO：纪要与行动项易丢失

  • 纪要整理耗时，难以结构化。
  • 行动项未分配负责人/截止时间。
  • 风险与依赖缺少提醒，跟进效率低。
后果

行动项执行率低，需要二次会议反复对齐。

02 · Flow

转写 → 纪要 → 行动项 → 提醒

录音/文本

收录录音或纪要草稿。

转写切片

ASR + 段落切分。

结构化纪要

议题/结论/决策点输出。

行动项

拆解负责人、DDL、依赖。

分配

同步到 JIRA/IM/日历。

提醒

定时提醒 + 风险提示。

02 · 节点演绎

Dify 节点与责任

转写入口

音频/文本输入，ASR/分段，保留时间戳。

纪要结构化

议程/决策/风险生成，引用原文链接。

同步与提醒

03 · 知识资产

模板、提示词、数据口径统一沉淀

模板库

Workflow 模板 + 触发器 + 输出格式，按部门/级别分类。

知识库

评测与监控

离线评测集 + 在线指标，验证一致性与安全。

03 · 运营

推广与治理的运营玩法

  • 模板市场：按场景上架/下架/版本回滚。
  • Owner 制：每条 Workflow 绑定责任人与指标。
  • 反馈闭环：用户反馈写回数据集，定期优化提示词/工具。

04 · 集成

连接 AISPEECH + 企业系统

  • 事件入口：定时、Webhook、消息、工单状态，统一进入 Dify。
  • 外部系统：JIRA、IM、邮件、语音/转写、BI、工单平台。
  • 回传路径：文档/IM/邮件推送、JIRA 更新、指标写回监控。
05 · 路线

4 周落地计划

  1. 第 1 周：选 2 个场景（信息同步官 + Bug Intake），定义口径与指标。
  2. 第 2-3 周：上线 Workflow + 模板市场内测；沉淀知识库与评测集。
  3. 第 4 周：扩展到多部门；建立运营节奏与治理规范。

06 · 风险

主要风险与缓解策略

  • 数据安全：采用私有化/专有云，敏感字段脱敏，按角色授权。
  • 口径漂移：版本化知识库 + 评测集，变更需通过评测。
  • 模型稳定性：缓存/多模型回退策略，关键链路加规则/校验。
  • 采纳度：模板市场 + 培训/运营机制，设置 Owner 与 KPI。
06 · FAQ

常见疑问

  • 能否复用现有系统？ 支持 API/Webhook，对接 JIRA、IM、语音/转写、BI。
  • 合规如何做？ 权限分级、审计日志、脱敏策略、模型黑白名单。
  • 如何控制成本？ 调用限额、缓存/分层、离线批处理替代实时。
  • 谁来维护？ 为每条 Workflow 绑定 Owner + 指标，运营节奏周迭代。
