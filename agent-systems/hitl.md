## Context

### Customer Messaging

The Human Input node represents a subtle but profound shift in how we think about AI applications.
Most current systems treat AI and humans as separate entities that hand work back and forth like runners in a relay race. This creates a fundamental disconnection: either the AI is working (and the human is waiting) or the human is working (and the AI is paused).
Dify sees this differently. We allow developers to embed highly configurable human touch points directly into the workflow's execution graph. It's not an external review process or a separate system - it's a first-class citizen in Dify's application architecture itself.


### Problem

The current generation of AI applications faces a fundamental tension: pure automation sacrifices quality control, while pure human operation sacrifices efficiency. Our customers consistently encounter three critical challenges:

1. Shifting Objectives Reality. Complex projects rarely follow a straight line from start to finish. Objectives evolve, contexts change, and priorities shift—especially in knowledge work. AI workflows excel at executing predefined steps but struggle with these mid-course adjustments. Without structured touchpoints for human intervention during execution, workflows become rigid and disconnected from emerging needs.
2. Trust Gap. Organizations remain hesitant to fully automate high-stakes processes, creating friction in AI adoption.
3. Integration Complexity. Existing HITL solutions like (go-to-human and Langgraph's interrupt) feel too clunky - they are unnecessarily technical, hard to visualize and maintain by non-technical team members, and create overheads to set up and integrate with Dify applications.

When customers express these problems, they often frame them through specific use cases:
1. Content teams request "approval action" for AI-generated content
2. Support teams ask for "human escalation" in chatbots
3. Platform teams want "form input" during ticket handling
### Solution

We're introducing a "Human Input" node for Dify's workflow canvas. This node serves as a pause point in any workflow where human judgment, review, or editing can be applied before execution continues. The node has 3 key components:

1. Delivery Configuration
  - A webpage will be generated by the node
  - Workflow developer can pick different channels to deliver the page to the human (webapp, Dify account email, Slack, etc. Leverages current plugins)
For simplicity, support for Dify plugins is left for later development. Currently input forms are delivered via Email.
2. Variable Referencing
  - Easily add workflow variables to the page via slash shortcut
  - Allow editing of these variables
  - Pass edited variables back to the workflow
3. Action Definition
  - Configurable action buttons
  - Automated resumption after human input
  - Custom action labels and routing to workflow paths based on human decisions
  - Timeout handling rules
This approach transforms collaboration from an external process into a native part of the application itself. Rather than building separate systems for human oversight, teams can now seamlessly integrate human touchpoints exactly where needed.

### Usage scenarios

Investment Services Team 

When Liang tries to deliver routine updates to their clients in May 2025, he faced a classic scaling problem. His team of junior analysts manually processed financial reports for institutional clients—a task consuming 40 minutes per client. With over 100 clients but limited staff, only the largest received proper service.
Liang built a Dify workflow that extracted data from turnsheets and generated client investment reports automatically. He placed Human Input nodes at three strategic points: after data summary and synthesis, when anomalous data appeared, and after report generation.
The compliance team could now review exactly what clients would see, make necessary edits, and approve with a single click for the report to be automatically sent to clients. By June, all 100 clients received consistent, high-quality reports.
"What's powerful is how I can thread human expertise through an otherwise automated process. The system handles the calculations, but humans apply judgment exactly where it matters most."

Global Support Team

Min inherited company's fragmented internal support system in June 2025. Employees navigated separate portals for HR, finance, and IT support—a structure reflecting organizational charts rather than employee needs.
She built an integrated workflow in Dify with a single entry point that classified queries and routed them to appropriate knowledge bases. But employee queries often lacked context, so Min implemented Human Input nodes to request specific information when needed.
When Jason from R&D asked about reimbursement in March, the system detected missing location data, prompted him through a Human Input node, then delivered precisely relevant information about Shanghai office procedures.
"We're building workflows that combine AI's consistency with human input's adaptability," Min explains. "The result feels like collaborative intelligence."
